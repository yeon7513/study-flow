> _Follow the Flow!_ <br />
> 공부의 흐름이 끊기지 않도록, 최적의 팀원과 기술을 잇다.

### 1️⃣ 주제

다양한 학습 및 특정 자격증 취득을 목표로 하는 사람들을 위한 팀원 매칭 기반 홍보 플랫폼의 출시 전, 얼마나 많은 사람이 관심을 가지는지 확인하고 이메일을 확보하는 것이 목표이다.

즉, 정식 출시 전 사용자의 수요와 데이터를 수집하고 분석하기 위한 웹사이트이다.

### 2️⃣ 컨셉

- 2030 타겟의 심플하고 직관적인 디자인으로 구현한다.
- 비회원 중심의 낮은 진입 장벽과 관리자 중심의 철저한 운영 관리를 목표로 한다.

## 🎓 주요 기능

### ⚙️ **관리자 인증 및 로그인**

- 일반 유저가 아닌 관리자만이 대시보드 접근을 위한 강력한 인증

### ⚙️ **스터디/매칭 사전 예약 시스템**

- 방문자가 관심 분야와 이메일을 남기는 핵심 홍보 기능

### ⚙️ **실시간 트렌드 렌더링**

- DB 집계 데이터를 활용한 인기 스터디 TOP 3 추천

### ⚙️ **비회원 Q&A 게시판**

- 진입 장벽을 낮춘 고객 소통 창구 및 서비스에 대한 문의 사항을 남기는 공간
- 회원가입을 강제하거나 사전 예약한 고객만 글을 작성할 수 있게 구현하는 것이 아닌 누구나 쉽게 접근할 수 있도록 비회원 게시판으로 운영
- 비회원 게시판이므로 무분별한 스팸 게시글을 통제하기 위해 신고 기능 구현

### ⚙️ **관리자 대시보드**

- 예약자 현황 파악
- Q&A 관리 기능
- 게시글 신고 처리 기능

## 📌 아키텍처 강점

React(화면)와 Spring Boot(데이터)를 완벽히 분리한 SPA 기반의 최신 RESTful API 설계를 적용한다.

### 🖥️ 프론트엔드

#### 1️⃣ 역할

- 서비스 정보 제공 및 사전 예약/Q&A 등록 폼을 구현한다.
- 관리자 대시보드를 통한 데이터 시각화 및 게시글 제어 UI를 제공한다.

#### 2️⃣ 사용자 친화적 UI/UX

- 백엔드에서 집계된 실시간 인기 분야 TOP 3를 메인 화면에 동적으로 렌더링 사용자 참여를 유도한다.
- 신고 완료 시 버튼 비활성화(Disabled) 처리 및 `LocalStorage`를 활용한 중복 액션 방지로 쾌적한 UX를 제공한다.

### 💽 백엔드

#### 1️⃣ 보안

- 단방향 암호화와 토큰(JWT/Session)을 활용한 관리자 인증 및 인가(Auth) 시스템을 구축한다.
- 백엔드에서 헤더에 담긴 토큰을 검사하여 role이 ADMIN인 경우에만 데이터를 반환한다.

#### 2️⃣ 유효성 검사 및 예외 처리

- 중복된 이메일 예약 시도 방지, 잘못된 형식의 데이터 차단 등 백엔드 단의 방어 로직을 설계한다.

#### 3️⃣ 성능 최적화

- 과거 Out of Memory 경험을 발판 삼아, 페이징(Pagination) 쿼리를 적용하여 서버 메모리 과부하를 원천 차단하는 구조로 설계한다.

#### 4️⃣ 데이터 집계 최적화

- 서버 메모리에서 랭킹을 계산하지 않고, SQL `GROUP BY`와 `LIMIT`을 활용해 연산 부하를 DB 계층으로 위임해 응답 속도를 최적화한다.
